# Topic Researcher — 领域词汇表

## 核心概念

### 技术问题 (Technical Question)
一个具体的、可回答的技术问题。例如："Rust 是否适合生产环境的 web 后端？"而不是"Rust"。

### 高置信度 (High Confidence)
一个观点被多个独立来源交叉印证，且至少有一个高信誉来源支持。不是单一声音，不是热门观点的简单重复。

### 共识 (Consensus)
≥3 个独立来源表达相同或高度相似的观点。独立性意味着不是互相转载，而是基于各自的经验或分析。

### 来源 Tier

**Tier-1（高信誉）：**
- 知名技术博客（如 Martin Fowler, Jamie Zawinski, Julia Evans）
- 官方文档和 RFC
- 有一手生产经验的工程师的详细博客文章
- 技术会议演讲和论文

**Tier-2（中等信誉）：**
- 活跃技术社区的高赞回答（HN frontpage, Reddit 高赞）
- 中型技术博客（有持续输出，作者身份可查）
- 技术新闻网站的深度分析

**Tier-3（低信誉）：**
- 匿名论坛评论
- 无作者署名的文章
- 低互动的个人博客
- SEO 内容农场

### 置信度评级

**High:**
- ≥3 个独立来源一致 + 至少 1 个 Tier-1 来源
- 无反面证据，或有反面证据但被明确驳斥

**Medium:**
- 2 个独立来源一致，或 ≥3 个来源但 tier 较低
- 或存在明显分歧但多数倾向一方

**Low:**
- 仅 1 个来源
- 或来源间严重分歧且无法判定哪方更可信

### 独立来源 (Independent Source)
不是互相转载、引用同一原始来源、或同一作者的不同文章。两个博客如果都引用同一个 Stack Overflow 回答，不算独立来源。

### 一手经验 (First-hand Experience)
作者直接参与过相关项目的开发和部署，而非评论他人的工作或转述二手信息。判断标准：文章是否包含具体的项目背景、技术细节、遇到的问题和解决方案。

### 反面证据 (Counter-evidence)
与主流观点相反的来源。不是所有不同意见都是反面证据——需要是基于经验或数据的有理有据的反驳，而不是单纯的偏好表达。

## 信源偏好

**优先级：**
1. 英文文献 > 中文文献
2. 技术社区讨论（HN, Reddit, X, LinkedIn）> 通用新闻媒体
3. 有一手经验的从业者 > 评论员/转载者
4. 高互动的高质量讨论 > 低互动的泛泛而谈

**高信号来源：**
- Hacker News 首页讨论
- Reddit 特定子版（r/programming, r/rust, r/golang 等）的高赞帖
- 知名工程师的 Twitter/X 线程
- LinkedIn 上一线工程师的详细文章
- 个人技术博客（如果有持续的高质量输出）

## 输出格式

结构化 Markdown 报告，包含：
- 核心发现（每个 claim 带 confidence level）
- 支持证据（来源 URL + 摘要）
- 共识度说明
- 反面证据（如果有）
- 总结
